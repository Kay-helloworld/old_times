# 資安申論題解析標準模板

> **用途**：此模板定義了申論題解析檔案的標準結構，適用於資訊安全、資料庫、資料結構等所有科目的考試準備。
>
> **使用方式**：當你想讓我以這個格式建立解析時，只需說：「請按照 **essay-analysis-template** 的格式處理」

---

## 📋 完整文件結構

### 1. 檔案標題與說明
```markdown
# [主題名稱] 完整題目解析 - 申論題答題框架

這份文件針對**[主題名稱]**相關題目提供詳盡的申論題答題架構，涵蓋[核心考點1]、[核心考點2]、[核心考點3]等核心考點。
```

### 3. 🎯 申論題答題黃金架構

```markdown
### 通用架構 (適用於[主題類型]題目)

第一部分：定義與背景 (15%)
├─ 定義關鍵術語
├─ 說明技術/概念的定位
└─ 闡述主要目的與重要性

第二部分：核心內容 (50%)
├─ 詳細說明運作原理/流程

├─ 列舉優點與缺點

第四部分：實務應用與延伸 (15%)
├─ 實際應用場景

└─ 解決方案或最佳實務
```

### 4. 📚 【核心知識】完全解析

**撰寫原則**：
- 提供關鍵概念的**完整定義**
- 使用**表格**進行對比說明
- 包含**公式、計算方法**（如適用）
- 提供**記憶口訣**輔助記憶

**範例結構**：
```markdown
### 一、[核心概念1] vs [核心概念2]

[使用表格對比]

### 二、[核心技術/方法] 詳解

[詳細說明 + 流程圖]

### 三、[常見分類/類型]

[分類表格 + 應用場景]
```

## 🎨 撰寫風格指南

### 答案示範的撰寫原則

1. **字數控制**：
   - 25分題：約800-1000字
   - 每小題10分：約300-400字
   - 避免過度冗長（實際考試時間限制）

2. **結構化**：
   - 使用編號（一、二、三）
   - 使用小標題
   - 使用列點說明

3. **專業性**：
   - 使用正確術語
   - 引用標準/規範
   - 提供數據支持

4. **實用性**：
   - 舉實際案例
   - 列具體產品/工具
   - 提供可操作建議

### 補充說明的撰寫原則

1. **豐富度**：
   - 真實案例（如：WannaCry勒索軟體）
   - 實務工具（如：Wireshark, nmap）
   - 相關標準（如：NIST, OWASP）
   - 延伸概念（如：新興技術）

2. **實用性**：
   - 指令範例（可直接執行）
   - 設定檔範例
   - 檢查清單

3. **可讀性**：
   - 表格對比
   - 流程圖
   - 記憶口訣

## ✅ 品質檢查清單

在完成每個解析檔案前，確認：

- [ ] 題型分類佔比表是否完整
- [ ] 每個題目是否有答題架構分析
- [ ] 每個題目是否有**配分建議**（在答題架構之後、作答之前）
- [ ] 配分建議是否包含**分數取捨策略**
- [ ] 標準答案字數是否適中（符合實際考試時間）
- [ ] 補充說明是否豐富（案例、工具、延伸）
- [ ] 是否有歷屆精選題庫
- [ ] 是否有答題技巧總結
- [ ] 是否有參考資源連結

## 📌 關鍵差異點：新版 vs 舊版

**保留自舊版**（使用者喜歡）：
- ✅ 題型分類佔比表
- ✅ 完整估分框架（每個配分項目都標示分數）
- ✅ 申論題回答框架（四部分結構）
- ✅ 核心知識說明（豐富的表格、公式、對比）
- ✅ 補充說明（案例、工具、延伸資源）

**改進自新版**（已註解掉的部分）：
- ⚠️ 擬答過於冗長 → 改為適中字數
- ⚠️ 缺乏補充說明 → 每題都有豐富補充

**新增功能**：
- ✨ **配分建議放在答題架構之後、作答之前**
- ✨ **配分建議包含分數取捨策略**（考場時間不夠時可優先捨棄哪些）
- ✨ **答題提示**（✅應做 / ⚠️避免）
