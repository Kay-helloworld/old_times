## 📚 核心知識速查

### 一、基本資料結構

```
【陣列 (Array)】
特性: 連續記憶體、固定大小
存取: O(1)
插入/刪除: O(n)

【鏈結串列 (Linked List)】
特性: 非連續、動態大小
存取: O(n)
插入/刪除: O(1) (已知位置)

【堆疊 (Stack)】
LIFO (後進先出)
操作: Push, Pop
應用: 函式呼叫、括號配對

【佇列 (Queue)】
FIFO (先進先出)
操作: Enqueue, Dequeue  
應用: 排程、BFS

【樹 (Tree)】
二元樹 (Binary Tree)
二元搜尋樹 (BST): 左<根<右
平衡樹: AVL, Red-Black
應用: 檔案系統、資料庫索引

【圖 (Graph)】
組成: 節點+邊
表示法:
├─ 相鄰矩陣 (Adjacency Matrix)
└─ 相鄰串列 (Adjacency List)
```

---

### 二、演算法複雜度

```
【時間複雜度 Big-O】
O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(2ⁿ) < O(n!)

【常見演算法】
├─ 搜尋
│   ├─ 線性搜尋: O(n)
│   └─ 二分搜尋: O(log n)
│
├─ 排序
│   ├─ 泡沫排序: O(n²)
│   ├─ 快速排序: O(n log n) 平均
│   └─ 合併排序: O(n log n)
│
└─ 圖形
    ├─ BFS/DFS: O(V+E)
    └─ Dijkstra: O(V²) or O(E log V)
```

---

###三、雜湊表 (Hash Table)

```
【概念】
鍵值對 (Key-Value)
透過雜湊函數對應到位置

【碰撞處理】
├─ 鏈結法 (Chaining)
└─ 開放定址法 (Open Addressing)

【時間複雜度】
平均: O(1)
最壞: O(n)
```

### 樹的遍歷

```
【前序 (Pre-order)】
根 → 左 → 右

【中序 (In-order)】
左 → 根 → 右

【後序 (Post-order)】
左 → 右 → 根

【層序 (Level-order)】
BFS逐層遍歷
```
