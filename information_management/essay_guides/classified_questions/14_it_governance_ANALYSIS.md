## 📚 核心知識速查

### 一、IT治理 (IT Governance)

```
【定義】
確保IT與業務目標一致
管理IT風險與資源

【COBIT框架】
Control Objectives for Information and Related Technologies

五大原則:
1. 滿足利害關係人需求
2. 涵蓋企業端到端
3. 應用單一整合框架
4. 啟用全方位方法
5. 區分治理與管理

【IT治理組成】
├─ IT策略委員會
├─ IT政策與程序
├─ IT績效管理
├─ 風險管理
└─ 法規遵循
```

---

### 二、ITIL (IT Service Management)

```
【定義】
IT Infrastructure Library
IT服務管理最佳實務框架

【服務生命週期】
1. 服務策略 (Service Strategy)
2. 服務設計 (Service Design)
3. 服務轉移 (Service Transition)
4. 服務營運 (Service Operation)
5. 持續服務改善 (Continual Service Improvement)

【重要流程】
├─ 事件管理 (Incident Management)
├─ 問題管理 (Problem Management)
├─ 變更管理 (Change Management)
├─ 服務台 (Service Desk)
└─ SLA管理
```

---

### 三、CMMI (能力成熟度模型整合)

```
【五個成熟度等級】
Level 1: 初始級 (Initial)
  - 混亂、臨機應變

Level 2: 管理級 (Managed)
  - 專案管理建立

Level 3: 定義級 (Defined)  
  - 流程標準化

Level 4: 量化管理級 (Quantitatively Managed)
  - 流程可量測

Level 5: 最佳化級 (Optimizing)
  - 持續改善

【目的】
評估與改善組織軟體流程成熟度
```

---

### 四、IT稽核

```
【稽核類型】
├─ 財務稽核: 會計系統正確性
├─ 營運稽核: 流程效率效果
├─ 法規遵循稽核: 符合法規
└─ 資訊系統稽核: IT控制有效性

【稽核步驟】
1. 規劃
2. 執行 (蒐集證據)
3. 評估
4. 報告
5. 追蹤改善

【常用框架】
├─ COBIT
├─ ISO 27001
└─ SOC 2 (System and Organization Controls)
```

---

### 五、組織績效管理

```
【BSC (平衡計分卡)】
四大構面:
1. 財務面
2. 顧客面  
3. 內部流程面
4. 學習成長面

【KPI設定】
SMART原則:
- Specific (具體)
- Measurable (可衡量)
- Achievable (可達成)
- Relevant (相關)
- Time-bound (有時限)
```

## 🎯 重點答題技巧

### IT治理 vs IT管理

| 面向 | IT治理 (Governance) | IT管理 (Management) |
|------|-------------------|-------------------|
| **層級** | 董事會/高階主管 | IT部門/中階主管 |
| **焦點** | 策略方向、價值創造 | 日常營運、執行 |
| **問題** | "做對的事" (Do the right things) | "做好這件事" (Do things right) |
| **責任** | 確保IT與業務一致 | 提供IT服務 |
| **時間** | 長期 | 短中期 |

---

### ITIL vs COBIT

| 面向 | ITIL | COBIT |
|------|------|-------|
| **焦點** | IT服務管理 | IT治理與管理 |
| **層級** | 營運層 | 治理層 |
| **用途** | 如何提供IT服務 | 如何治理IT |
| **範圍** | IT服務生命週期 | 整體IT |
