## 📚 核心知識解析

### 一、5G 第五代行動通訊技術

#### (1) 5G 定義與演進

**英文全名**: **5th Generation mobile networks** (第五代行動網路)

**定義**: 5G是**第五代行動通訊技術標準**,由3GPP (3rd Generation Partnership Project) 制定,相較於4G提供**更高速度、更低延遲、更大連接密度**,支援**增強型行動寬頻 (eMBB)**、**大規模物聯網 (mMTC)** 及**關鍵任務通訊 (URLLC)**。

**行動通訊演進史**:

```
1G (1980s): 類比語音通話
    ↓
2G (1990s): 數位語音 + 簡訊 (GSM)
    ↓
3G (2000s): 行動上網 (3.5Mbps)
    ↓
4G/LTE (2010s): 高速上網 (100Mbps-1Gbps)
    ↓
5G (2020s): 超高速+低延遲+大連接 (10Gbps+)
```

---

#### (2) 5G 三大技術場景

**場景1: eMBB (Enhanced Mobile Broadband, 增強型行動寬頻)**

**定義**: 提供**超高速率**行動寬頻服務

**關鍵指標**:
- 峰值速率: **20 Gbps** (下載), **10 Gbps** (上傳)
- 實際體驗: **100 Mbps - 1 Gbps** (4G約10-50 Mbps)

**應用場景**:
```
✅ 4K/8K超高畫質影音串流
✅ VR/AR沉浸式體驗 (需大頻寬)
✅ 雲端遊戲 (Cloud Gaming)
✅ 高解析度視訊會議
```

**實例**:
```
【4K影片下載】
- 4G: 1小時電影 (10GB) → 下載需15分鐘
- 5G: 同樣電影 → 下載僅需10秒!

【VR直播】
- VR需每秒傳輸100Mbps以上 (雙眼各50Mbps)
- 4G: 延遲+頻寬不足 → 暈眩
- 5G: 流暢體驗
```

---

**場景2: mMTC (Massive Machine Type Communication, 大規模機器通訊)**

**定義**: 支援**大量IoT設備**同時連線

**關鍵指標**:
- 連接密度: **100萬設備 / 平方公里**
- 4G僅支援約**10萬設備 / 平方公里**

**特性**:
- 低功耗: 電池可用**10年以上** (NB-IoT)
- 小數據量: 設備定期回報少量資料
- 低成本: 模組價格低

**應用場景**:
```
✅ 智慧城市: 數萬個路燈、垃圾桶、停車格感測器
✅ 智慧農業: 大面積農田溫濕度監測
✅ 智慧電表: 每戶都是IoT節點
✅ 資產追蹤: 物流倉庫數萬件貨物RFID
```

**實例: 智慧城市路燈**
```
傳統: 全開或全關
      ↓
智慧化 (每盞路燈都是IoT節點):
├─ 感測器: 偵測人車經過
├─ 5G連線: 即時回報
├─ AI決策: 有人才開燈 (省電70%)
└─ 數量: 一個城市數萬盞 → 需mMTC支援
```

---

**場景3: URLLC (Ultra-Reliable Low-Latency Communication, 超高可靠低延遲通訊)**

**定義**: 提供**超低延遲**及**高可靠性**通訊

**關鍵指標**:
- 延遲: **< 1ms** (毫秒, 0.001秒)
- 4G延遲: 約30-50ms
- 可靠性: **99.999%** (Five Nines)

**為何需要超低延遲?**
```
【自駕車】
車速100km/h = 27.8 m/s

延遲50ms (4G) → 煞車前車子已前進1.4公尺 → 可能撞上
延遲1ms (5G) → 煞車前僅前進2.78公分 → 安全!

【遠端手術】
醫生在台北操作機器手臂,為高雄病人開刀
延遲50ms → 手術刀位置誤差 → 危險!
延遲1ms → 即時同步 → 安全
```

**應用場景**:
```
✅ 自駕車 (車聯網 V2X)
✅ 遠端醫療手術
✅ 工業自動化 (智慧工廠機器人)
✅ 無人機控制
✅ 關鍵基礎設施 (電網、交通號誌)
```

**實例: 自駕車V2X**
```
V2V (Vehicle-to-Vehicle): 車與車通訊
├─ 前車急煞 → 1ms內通知後車 → 自動煞車
└─ 避免追撞

V2I (Vehicle-to-Infrastructure): 車與基礎設施
├─ 紅綠燈倒數 → 車輛接收 → 調整車速
└─ 減少等待時間

V2P (Vehicle-to-Pedestrian): 車與行人
├─ 偵測行人手機 → 預警可能穿越馬路
└─ 避免撞到低頭族
```

#### (3) 5G vs 4G 完整比較

| 面向 | 4G LTE | 5G | 提升倍數 |
|------|--------|----|----|
| **峰值速率** | 1 Gbps | 20 Gbps (下載) | **20倍** |
| **實際體驗** | 10-50 Mbps | 100 Mbps - 1 Gbps | **10-100倍** |
| **延遲** | 30-50 ms | < 1 ms | **30-50倍** |
| **連接密度** | 10萬/km² | 100萬/km² | **10倍** |
| **頻譜效率** | 基準 | 3倍 | **3倍** |
| **移動性** | 350 km/h | 500 km/h | 支援高鐵 |
| **能源效率** | 基準 | 100倍 | **省電** |
| **流量密度** | 0.1 Tbps/km² | 10 Tbps/km² | **100倍** |

**技術差異**:

| 技術 | 4G | 5G |
|------|----|----|
| **頻段** | 低頻段 (< 6 GHz) | 低頻+中頻+**毫米波 (24-100 GHz)** |
| **天線技術** | MIMO (4x4) | **Massive MIMO (64x64, 256x256)** |
| **核心網路** | EPC (演進封包核心) | **5GC (雲原生、網路切片)** |
| **調變技術** | OFDM | **CP-OFDM + DFT-S-OFDM** |
| **波束成形** | 基本 | **先進波束成形** (精準對準設備) |

#### (4) 5G 與 IoT 的關聯性 (重要考題!)

**問**: 為何5G對IoT如此重要?

**答**: 5G的三大特性**完美對應**IoT的三大需求:

```
【IoT需求1: 大量設備連線】
├─ 智慧城市數萬感測器
├─ 智慧工廠數千機器人
└─ 5G解法: mMTC (100萬設備/km²)

【IoT需求2: 即時反應】
├─ 自駕車需即時決策
├─ 工業機器人需精準控制
└─ 5G解法: URLLC (< 1ms延遲)

【IoT需求3: 大量數據傳輸】
├─ 監控攝影機4K串流
├─ VR/AR應用
└─ 5G解法: eMBB (20 Gbps)
```

**5G賦能IoT應用矩陣**:

| IoT應用 | 需要的5G特性 | 為何需要5G |
|---------|------------|----------|
| **智慧城市** | mMTC | 數萬路燈、停車格、垃圾桶感測器 |
| **智慧醫療** | URLLC | 遠端手術需< 1ms延遲 |
| **智慧工廠** | URLLC + mMTC | 機器人需低延遲+大量設備連線 |
| **自駕車** | URLLC + eMBB | 煞車需< 1ms + 3D地圖需大頻寬 |
| **智慧家庭** | mMTC | 數十個IoT設備 (電燈、冷氣、門鎖) |
| **智慧農業** | mMTC | 大面積農田感測器 |
| **AR/VR** | eMBB + URLLC | 大頻寬+低延遲避免暈眩 |

**案例: 5G+IoT 智慧工廠**

```
【傳統工廠】
├─ 有線連接 (布線複雜、不易調整)
├─ 人工巡檢 (效率低)
└─ 故障後才維修 (損失大)

【5G智慧工廠】
├─ 無線連接: 5G取代有線 (彈性高)
├─ IoT感測器: 即時監控機器狀態 (mMTC)
├─ AI預測: 預測性維護 (URLLC低延遲回傳)
├─ AGV (自動搬運車): 5G導航 (< 1ms)
└─ AR輔助: 工程師戴AR眼鏡維修 (eMBB)

【效益】
✅ 生產效率 ↑30%
✅ 故障率 ↓50%
✅ 人力成本 ↓40%
```

### 二、物聯網 (IoT) 架構與應用

#### (1) IoT 定義

**物聯網 (Internet of Things, IoT)**: 透過**感測器、RFID、網路通訊**等技術,將**實體物品連接到網際網路**,實現物品之間、物品與人之間的**資訊交換與智慧化管理**。

**核心概念**:
- **萬物互聯**: 任何物品都可連網 (不只電腦、手機)
- **智慧感知**: 透過感測器蒐集環境資料
- **資料驅動**: 分析資料並做出智慧決策
- **自動化**: 減少人工介入

**IoT vs 傳統網際網路**:

| 面向 | 傳統網際網路 | 物聯網 (IoT) |
|------|------------|------------|
| 連接對象 | 人 (透過電腦、手機) | **物品** (任何東西) |
| 連接數量 | 數十億 | **數千億** (2025年預估500億) |
| 主要功能 | 資訊傳遞 | **感知+傳遞+控制** |
| 互動方式 | 人操作 | **自動化** (M2M, Machine-to-Machine) |
| 範例 | 上網瀏覽、Email | 智慧手環、智慧電表、自駕車 |

---

#### (2) IoT 三層架構 (重要必考!)

IoT系統可分為**感知層、網路層、應用層**三層架構:

```
┌─────────────────────────────────────┐
│      應用層 (Application Layer)      │
│  ├─ 智慧城市、智慧醫療、智慧家庭    │
│  ├─ 大數據分析、AI決策              │
│  └─ 應用服務平台                    │
└────────────┬────────────────────────┘
             ↑↓ (資料處理與決策)
┌─────────────────────────────────────┐
│       網路層 (Network Layer)         │
│  ├─ 傳輸技術: WiFi, 4G/5G, LoRa     │
│  ├─ 閘道器 (Gateway)                │
│  └─ 雲端平台                        │
└────────────┬────────────────────────┘
             ↑↓ (資料傳輸)
┌─────────────────────────────────────┐
│       感知層 (Perception Layer)      │
│  ├─ 感測器 (溫溼度、光線、動作)     │
│  ├─ RFID / NFC                      │
│  ├─ 攝影機、麥克風                  │
│  └─ 致動器 (Actuator, 如馬達)       │
└─────────────────────────────────────┘
```

**層次1: 感知層 (Perception Layer)**

**功能**: **感知、識別、控制**實體世界

**組成元件**:

| 元件類型 | 功能 | 範例 |
|---------|------|------|
| **感測器 (Sensor)** | 蒐集環境資料 | 溫濕度、PM2.5、光線、聲音、加速度 |
| **RFID標籤** | 物品識別 | 商品標籤、門禁卡 |
| **NFC** | 近距離通訊 | 行動支付、門禁 |
| **攝影機** | 影像辨識 | 人臉辨識、車牌辨識 |
| **GPS** | 定位 | 物流追蹤、導航 |
| **致動器 (Actuator)** | 執行動作 | 馬達、閥門、電燈開關 |

**技術**:
- **RFID (Radio Frequency Identification)**: 無線射頻識別
- **WSN (Wireless Sensor Network)**: 無線感測網路
- **二維碼 / QR Code**: 物品標識

**實例: 智慧農業**
```
感知層設備:
├─ 土壤溼度感測器
├─ 溫度感測器
├─ 光照感測器
├─ 攝影機 (監控作物)
└─ 噴灌閥門 (致動器)
```

---

**層次2: 網路層 (Network Layer)**

**功能**: **傳輸資料**從感知層到應用層

**組成元件**:

| 元件 | 功能 | 技術 |
|------|------|------|
| **閘道器 (Gateway)** | 協議轉換、資料聚合 | 將感測器資料轉成IP協議 |
| **傳輸技術** | 資料傳輸 | WiFi, 4G/5G, LoRa, NB-IoT, Zigbee |
| **邊緣運算** | 本地處理 | Edge Computing (減輕雲端負擔) |
| **雲端平台** | 資料儲存與運算 | AWS IoT, Azure IoT Hub |

**IoT常用傳輸技術比較**:

| 技術 | 距離 | 功耗 | 速率 | 適用場景 |
|------|------|------|------|---------|
| **WiFi** | 100m | 高 | 高 (Mbps) | 智慧家庭 |
| **Bluetooth** | 10-100m | 低 | 中 (Mbps) | 穿戴裝置 |
| **Zigbee** | 10-100m | 極低 | 低 (Kbps) | 智慧家庭感測網 |
| **LoRa** | 15km | 極低 | 低 (Kbps) | 智慧農業、城市 |
| **NB-IoT** | 全國 | 極低 | 低 (Kbps) | 智慧電表、追蹤 |
| **4G/5G** | 全國 | 中高 | 極高 (Gbps) | 車聯網、監控 |

**閘道器角色**:
```
【問題】
感測器使用Zigbee,但雲端平台用HTTP/IP
→ 協議不相容!

【解法: Gateway】
Zigbee感測器 → Gateway (協議轉換) → WiFi → Internet → 雲端

Gateway功能:
├─ 協議轉換 (Zigbee ↔ IP)
├─ 資料聚合 (100個感測器資料打包)
├─ 邊緣運算 (初步過濾、異常偵測)
└─ 安全閘道 (加密)
```

---

**層次3: 應用層 (Application Layer)**

**功能**: 提供**智慧化應用服務**

**組成**:
- **應用平台**: 提供API、儀表板、規則引擎
- **大數據分析**: 處理海量IoT資料
- **AI決策**: 機器學習預測、異常偵測
- **應用服務**: 各種垂直領域應用

**IoT應用領域**:

| 領域 | 應用 | 技術 |
|------|------|------|
| **智慧家庭** | 智慧音箱、自動窗簾、安全監控 | WiFi, Zigbee |
| **智慧城市** | 智慧路燈、停車管理、垃圾監測 | LoRa, NB-IoT |
| **智慧醫療** | 遠距照護、穿戴裝置、藥物追蹤 | Bluetooth, 4G |
| **智慧交通** | 車聯網、智慧號誌、停車導引 | 5G, V2X |
| **智慧工廠** | 設備監控、預測性維護、AGV | 5G, WiFi |
| **智慧農業** | 溫室監控、灌溉自動化、畜牧管理 | LoRa, NB-IoT |
| **智慧零售** | 無人商店、庫存管理、客流分析 | RFID, 攝影機 |
| **智慧能源** | 智慧電表、太陽能監控、需量管理 | NB-IoT |

### 三、IoT 安全 (重要考題!)

#### (1) IoT 主要安全威脅

**威脅1: 設備安全漏洞**

```
【問題】
├─ 預設密碼: admin/admin (從不更改)
├─ 無法更新: 韌體太舊,有已知漏洞
├─ 弱加密: 甚至明文傳輸
└─ 無防護: 無防火牆

【實際案例: Mirai殭屍網路 (2016)】
├─ 攻擊對象: IoT攝影機、路由器
├─ 方法: 掃描預設密碼設備
├─ 結果: 組成60萬台殭屍網路
└─ 發動DDoS: 癱瘓DNS服務商Dyn
    → Twitter, Netflix, Spotify全當機
```

**威脅2: 通訊劫持**

```
【中間人攻擊 (MITM)】
智慧門鎖 ←→ [黑客] ←→ 手機App
         ↑
      攔截開門指令 → 破解密碼

【重送攻擊 (Replay Attack)】
黑客錄下「開燈」指令 → 重送 → 燈無限開關
```

**威脅3: 隱私侵犯**

```
【RFID追蹤】
├─ 悠遊卡RFID被讀取 → 知道你的行蹤
├─ 商品RFID未消磁 → 購物習慣被追蹤
└─ 解法: RFID屏蔽卡套、離店消磁

【智慧音箱竊聽】
├─ 7x24小時開著麥克風
├─ 可能未經同意蒐集對話
└─ 解法: 實體靜音鍵、審查隱私政策
```

**威脅4: 殭屍網路 (Botnet)**

```
黑客攻陷大量IoT設備 → 組成botnet
↓
發動DDoS攻擊 (Distributed Denial of Service)
├─ 目標: 政府網站、企業
└─ 結果: 癱瘓服務
```

#### (2) IoT 攻擊原理與案例

**案例: 智慧汽車駭客入侵**

```
【攻擊流程】
1. 漏洞掃描
   ├─ 車載娛樂系統有漏洞
   └─ 透過3G/4G連線可遠端存取

2. 初步入侵
   ├─ 遠端連線到娛樂系統
   └─ 執行惡意程式碼

3. 橫向移動
   ├─ 娛樂系統與CAN Bus相連 (車內網路)
   └─ 透過CAN Bus控制引擎、煞車

4. 完全控制
   ├─ 遠端控制車輛加速、煞車、轉向
   └─ 極度危險!

【實際案例: 2015年Jeep Cherokee】
├─ 安全研究員Charlie Miller遠端控制Jeep
├─ 證明可透過車載系統入侵
└─ 結果: Chrysler召回140萬輛車
```

#### (3) IoT 防範措施 (分層防護)

**設備層防護**:

| 措施 | 說明 |
|------|------|
| **強制改密碼** | 首次啟用強制更改預設密碼 |
| **定期更新** | 自動推送韌體更新 (OTA, Over-The-Air) |
| **最小權限** | 設備僅開放必要功能 |
| **安全啟動** | Secure Boot防止韌體被竄改 |
| **TPM晶片** | 硬體加密模組 |
| **物理防護** | 防拆設計,拆開即失效 |

**網路層防護**:

```
【網路隔離】
IoT設備 (VLAN 1) ←→ 防火牆 ←→ 企業網路 (VLAN 2)
                        ↑
                   僅允許必要流量

【加密傳輸】
├─ TLS 1.3: 傳輸層加密
├─ DTLS: UDP加密 (CoAP協議)
└─ 避免明文傳輸

【異常偵測】
├─ 監控設備行為
├─ AI偵測異常流量 (如攝影機突然大量對外連線)
└─ 自動隔離可疑設備
```

**應用層防護**:

```
【身分認證】
├─ 多因素認證 (MFA)
├─ OAuth 2.0授權
└─ JWT (Json Web Token)

【存取控制】
├─ RBAC (Role-Based Access Control)
├─ 最小權限原則
└─ 定期審查權限

【資料加密】
├─ 靜態加密 (AES-256)
├─ 資料庫欄位加密
└─ 去識別化 (匿名化)
```

**管理層防護**:

```
【政策制定】
├─ IoT安全政策
├─ 禁用預設密碼
└─ 強制定期更新

【監控與稽核】
├─ SIEM (Security Information and Event Management)
├─ 日誌集中管理
└─ 定期漏洞掃描

【事件應變】
├─ 建立CSIRT (資安事件應變小組)
├─ 演練入侵情境
└─ 備援機制
```

### 技巧五: 常見陷阱

**陷阱1: 混淆概念**
- ❌ IoT ≠ M2M (M2M較狹義,IoT更廣)
- ❌ RFID ≠ NFC (NFC距離更短,雙向通訊)
- ❌ LoRa ≠ NB-IoT (LoRa非電信商,NB-IoT是)

**陷阱2: 只提技術,沒說效益**
- ❌ 「智慧路燈用IoT」
- ✅ 「智慧路燈用PIR感測器偵測人車,僅在需要時開燈,省電70%」

**陷阱3: 忽略安全**
- IoT題目一定要提安全考量!
