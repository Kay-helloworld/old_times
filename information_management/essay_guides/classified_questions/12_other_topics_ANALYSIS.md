## 📚 核心知識解析

### 一、資訊委外管理 (IT Outsourcing)

#### (1) 資訊委外定義

**資訊委外 (IT Outsourcing)**: 組織將部分或全部**資訊相關業務**委託給**外部專業廠商**執行,以降低成本、取得專業、聚焦核心業務。

**委外範圍**:

```
【可委外項目】
├─ 基礎設施管理
│   ├─ 機房管理
│   ├─ 網路維護
│   ├─ 伺服器管理
│   └─ 桌面支援
│
├─ 應用系統開發
│   ├─ 客製化開發
│   ├─ 系統維護
│   └─ 系統整合
│
├─ 資安服務
│   ├─ 弱點掃描
│   ├─ 滲透測試
│   ├─ 資安監控 (SOC)
│   └─ 事件應變
│
└─ 雲端服務
    ├─ IaaS (基礎設施)
    ├─ PaaS (平台)
    └─ SaaS (軟體)

【不適合委外】
❌ 核心業務系統 (競爭優勢來源)
❌ 高度機密資料
❌ 需高度客製化且頻繁變更
❌ 法規禁止委外項目
```

#### (2) 委外決策因素 (114年高等二等考題!)

**決策考量**:

**因素1: 成本效益分析**

```
【成本比較】
自建成本:
├─ 人力成本 (薪資、訓練)
├─ 設備成本 (硬體、軟體)
├─ 維護成本
└─ 機會成本

委外成本:
├─ 委外費用
├─ 管理成本 (監督)
├─ 轉換成本
└─ 風險成本

【經濟規模】
委外廠商服務多個客戶
→ 分攤成本
→ 可能比自建便宜

【範例】
中小企業自建機房:
├─ 購置伺服器: $500萬
├─ 聘請3位MIS: $200萬/年
└─ 機房維護: $50萬/年

委外雲端:
├─ AWS年費: $100萬
└─ 節省$650萬!
```

---

**因素2: 專業能力與技術**

```
【技術需求】
組織需要但缺乏的技術:
├─ 新興技術 (AI, 區塊鏈)
├─ 專業技能 (資安、大數據)
└─ 特定領域知識

【廠商優勢】
委外廠商:
✅ 專業團隊
✅ 豐富經驗
✅ 最新技術
✅ 最佳實務

【範例】
政府機關需開發AI系統:
自建: 缺AI人才、經驗
委外: 專業AI公司,有成功案例
→ 決定委外
```

---

**因素3: 核心業務聚焦**

```
【資源聚焦】
將IT交給專業廠商
→ 組織專注核心業務

【範例】
銀行:
核心業務: 金融服務、風險管理
非核心: IT基礎設施

策略:
委外IT基礎設施 (機房、網路)
自建金融核心系統 (交易、風控)
```

---

**因素4: 風險與控制**

```
【風險考量】
委外風險:
├─ 廠商倒閉
├─ 服務品質不穩
├─ 資料外洩
├─ 依賴性 (難以轉換)
└─ 失去內部能力

控制需求:
├─ 需掌控的系統 → 自建
└─ 可接受風險的 → 委外

【範例】
核心交易系統:
風險: 停機損失巨大
控制: 需完全掌控
→ 不委外

客服系統:
風險: 可接受
控制: 不需完全掌控
→ 可委外
```

#### (3) 資訊委外4面向利益 (114年高等二等完整題!)

**利益1: 經濟面 (Economic)**

```
【成本節省】
直接節省:
├─ 降低人力成本 (不需聘全職IT人員)
├─ 減少設備投資 (硬體、軟體授權)
├─ 節省維護費用
└─ 避免技術過時損失

間接節省:
├─ 提高效率
├─ 加速上線時間 (Time-to-Market)
└─ 避免錯誤成本

【彈性付費】
按需付費 (Pay-as-you-go)
├─ 雲端服務: 用多少付多少
├─ 避免閒置浪費
└─ 高峰/離峰彈性調整

【資本支出→營運支出】
CapEx (Capital Expenditure) → OpEx (Operational)
├─ 自建: 一次大筆投資 (CapEx)
└─ 委外: 分期付費 (OpEx)

優勢: 改善現金流、降低財務風險

【範例】
企業自建vs委外成本比較:

自建 (3年):
├─ 初期投資: $1,000萬 (CapEx)
├─ 年維護: $200萬/年 × 3 = $600萬
└─ 總成本: $1,600萬

委外 (3年):
├─ 年費: $400萬/年 × 3 = $1,200萬 (OpEx)
└─ 節省: $400萬 (25%)
```

---

**利益2: 管理面 (Managerial)**

```
【聚焦核心業務】
釋放管理階層時間與心力
├─ IT交給專業廠商
└─ 專注策略與核心業務

【簡化管理】
減少直接管理負擔:
├─ 不需管IT團隊
├─ 不需技術決策 (硬體選型等)
└─ 廠商負責人員管理

【風險分擔】
委外廠商承擔部分風險:
├─ 技術風險 (系統故障)
├─ 人員風險 (離職)
└─ 合約定義責任歸屬

【服務水準保證 (SLA)】
Service Level Agreement
├─ 明確定義服務標準
├─ 系統可用性 (如99.9%)
├─ 回應時間
└─ 未達標→罰則

效益: 可預測的服務品質

【範例】
政府機關委外機房管理:

自管困難:
├─ 需專業機房人員 (難招募)
├─ 24/7輪班 (人力吃緊)
├─ 技術更新快 (培訓成本高)

委外優勢:
✅ 廠商有專業團隊
✅ 24/7監控服務
✅ SLA保證 (如99.95%可用性)
✅ 政府專注公共服務
```

---

**利益3: 技術面 (Technical)**

```
【取得專業技術】
委外廠商優勢:
├─ 專業團隊 (如資安專家)
├─ 最新技術掌握
├─ 豐富專案經驗
└─ 業界最佳實務

組織獲益:
✅ 快速取得先進技術
✅ 不需自行培養 (省時間)
✅ 避免技術選擇錯誤

【技術更新**
IT技術快速演進
├─ 硬體: 3-5年淘汰
├─ 軟體: 持續更新
└─ 自建: 需持續投資

委外:
✅ 廠商負責技術更新
✅ 始終使用最新技術
✅ 避免技術債

【技術轉移**
委外過程學習:
├─ 廠商分享知識
├─ 教育訓練
└─ 內部能力提升

【擴充性與彈性】
雲端委外優勢:
├─ 彈性擴充 (分鐘級)
├─ 全球部署
└─ 災難復原能力

自建限制:
❌ 擴充需採購 (數月)
❌ 受限機房空間
❌ 異地備援成本高

【範例】
新創公司開發App:

委外雲端 (AWS):
✅ 初期小流量: 低成本
✅ 爆紅時: 快速擴充
✅ 全球CDN加速
✅ 自動備份、容災

自建:
❌ 前期投資大
❌ 預估流量困難
❌ 擴充慢
```

```

   │
   ├─ RFI (Request for Information, 資訊邀請書)
   │   └─ 市場調查、了解廠商能力
   │
   └─ RFQ (Request for Quotation, 報價邀請書)
       └─ 已知需求,僅需報價

2. 資安需求 (重要!)

   ├─ 受託者環境要求
   ├─ 人員安全要求 (背景調查)
   ├─ 資料保護措施
   ├─ 事件通報機制
   └─ 複委託限制

3. 公告招標
   ├─ 政府採購法 (政府機關)
   │   ├─ 公開招標
   │   ├─ 選擇性招標
   │   └─ 限制性招標
   └─ 廠商資格審查

4. 廠商提案
   └─ 接收廠商企劃書、報價

【關鍵產出】
✅ RFP需求文件
✅ 招標公告
✅ 廠商提案(收件)

【注意事項】
⚠️ RFP需求明確,避免模糊
⚠️ 資安需求務必詳載
⚠️ 評選標準客觀、可量化
⚠️ 留意採購法規
```

```
   ├─ 智慧財產權歸屬
   └─ 保密條款

3. 決標
   └─ 簽訂正式合約

【關鍵產出】
✅ 評選報告
✅ 正式合約

【合約重點條款】
📋 範疇與交付項目
📋 時程與里程碑
📋 費用與付款方式
📋 SLA與罰則
📋 智財權歸屬
📋 保密義務
📋 資安責任
📋 變更管理程序
📋 爭議處理機制
📋 終止條款

【注意事項】
⚠️ 合約需法務審查
⚠️ SLA要能量化、可監控
⚠️ 罰則要有嚇阻力
⚠️ 預留彈性 (變更管理)
```

```
【主要工作】

1. 專案管理
   ├─ 成立專案管理辦公室 (PMO)
```

### 二、資訊倫理與法規 (Information Ethics)

#### (1) 資訊倫理定義

**資訊倫理**: 資訊時代中處理**資訊蒐集、處理、傳播**所涉及的**道德準則與行為規範**,探討資訊科技對個人、組織、社會的影響,以及相關的**權利與責任**。

**為何重要?**

```
【資訊時代特性】
├─ 資訊易於複製 (零邊際成本)
├─ 傳播速度快 (病毒式傳播)
├─ 影響範圍廣 (全球化)
└─ 難以消除 (網路記憶)

【倫理挑戰】
傳統倫理規範 vs 新型態問題
├─ 個資侵犯
├─ 數位監控
├─ 假新聞
├─ AI偏見
└─ 數位落差
```

---

#### (2) PAPA模式 (114年高等二等完整題!)

**PAPA**: R.O. Mason (1986) 提出資訊倫理四大議題框架

```
P - Privacy (隱私權)
A - Accuracy (正確性)
P - Property (所有權)
A - Accessibility (近用權)
```

---

**P1: Privacy (隱私權)**

**定義**: 個人**控制自己資訊**的權利,決定誰可以蒐集、使用、分享個人資料。

**核心議題**:

```
【個資蒐集】
組織大量蒐集個資:
├─ 網站: 瀏覽行為、Cookie
├─ App: 位置、通訊錄、相機
├─ 社群: 按讚、留言、分享
└─ 物聯網: 穿戴裝置健康資料

問題: 
❌ 過度蒐集 (超出必要範圍)
❌ 未充分告知
❌ 同意非自願 (不同意就不給用)

【個資利用】
次級利用 (Secondary Use):
原本為A目的蒐集
→ 用於B目的 (如轉賣)

範例:
購物網站蒐集地址 (為了寄貨)
→ 賣給廣告商 (未經同意)

【個資外洩】
案例:
├─ 2023 MOVEit漏洞: 全球數千企業個資外洩
├─ 2021 Facebook: 5億用戶資料外洩
├─ 2020 台灣戶政系統遭駭
└─ 2017 Equifax: 1.4億美國人信用資料外洩

影響:
💔 身分盜用
💔 詐騙
💔 信譽損害

【數位監控】
政府/企業監控:
├─ 監視器+人臉辨識
├─ 網路監控
├─ 員工監控 (鍵盤、螢幕)
└─ 社群媒體監看

爭議: 安全 vs 隱私
```

**相關法規**:

```
【台灣】
個人資料保護法 (個資法, 2012)
├─ 規範個資蒐集、處理、利用
├─ 當事人權利: 查詢、更正、刪除
├─ 違反罰則: 最高2億元

【歐盟】
GDPR (General Data Protection Regulation, 2018)
├─ 被遺忘權 (Right to be Forgotten)
├─ 資料可攜權
├─ 違反罰款: 營收4%或2千萬歐元

【國際】
隱私權保護原則:
├─ 通知與同意 (Notice & Consent)
├─ 最小化蒐集 (Data Minimization)
├─ 目的限制 (Purpose Limitation)
├─ 安全保護 (Security Safeguards)
└─ 責任歸屬 (Accountability)
```

---

**A1: Accuracy (正確性)**

**定義**: 資訊的**真實性、準確性、完整性**,以及對錯誤資訊的**責任歸屬**。

**核心議題**:

```
【假新聞 (Fake News) - 112年高考考題!】

定義:
刻意製造的虛假新聞,意圖誤導

特徵:
├─ 聳動標題 (Clickbait)
├─ 無可靠消息來源
├─ 時間地點模糊
├─ 違反常識
└─ 情緒煽動

範例:
「XX食品含致癌物」(未經證實)
→ 引起恐慌,該品牌股價暴跌

【錯誤訊息 (Misinformation)】

定義:
非刻意的錯誤資訊

範例:
網友誤傳疫情謠言 (本人相信是真的)

【假消息 (Disinformation)】

定義:
刻意製造並傳播的虛假資訊

目的:
├─ 政治操控 (影響選舉)
├─ 經濟利益 (操縱股市)
├─ 社會分裂
└─ 外國資訊戰

【AI生成假資訊】
ChatGPT等AI可能產生:
├─ 虛假但看似可信的內容
├─ 編造來源
└─ 「AI幻覺」(Hallucination)

【三者比較】
假新聞 (Fake News):
  刻意+新聞形式

錯誤訊息 (Misinformation):
  非刻意,誤傳

假消息 (Disinformation):
  刻意,任何形式
```

**影響**:

```
【對民主制度】
├─ 操控選舉 (假消息抹黑候選人)
├─ 削弱信任 (人民不信媒體、政府)
├─ 社會撕裂 (製造對立)
└─ 理性討論困難

【對公眾信任】
├─ 媒體可信度下降
├─ 專家意見被質疑
├─ 「後真相時代」(Post-truth)
└─ 陰謀論盛行

【實際危害】
案例:
├─ 2020美國大選爭議
├─ COVID-19疫苗假訊息 → 拒打疫苗
├─ 2016英國脫歐假資訊
└─ 台灣選舉假訊息攻擊
```

**應對策略** (112年高考考題!):

```
【政府角色】
1. 立法規範
   ├─ 台灣: 散布假訊息可依社維法處罰
   ├─ 但需平衡言論自由
   └─ 避免成為言論審查

2. 成立事實查核單位
   ├─ 台灣事實查核中心
   └─ 快速澄清假訊息

3. 資訊素養教育
   ├─ 培養批判思考
   └─ 查證能力

【媒體責任】
1. 自律機制
   ├─ 新聞倫理
   ├─ 查證後再報導
   └─ 更正錯誤報導

2. 標示來源
   └─ 清楚註明消息來源

【社交平台】
1. 標註可疑內容
   ├─ Facebook: 「獨立事實查核員認為此訊息不實」
   └─ Twitter/X: 加註「背景資訊」

2. 降低假訊息觸及
   └─ 演算法調整

3. 移除極端內容
   └─ 煽動暴力、仇恨言論

【公民社會組織】
1. 事實查核組織
   ├─ Cofacts真的假的
   ├─ MyGoPen
   └─ 台灣事實查核中心

2. 媒體識讀教育
   └─ 教導辨識假新聞

【個人】
辨識假新聞方法:
✅ 查證來源 (是否為可信媒體)
✅ 比對多個來源
✅ 檢查日期 (是否舊聞)
✅ 看完整內文 (不只標題)
✅ 圖片反搜 (Google圖片搜尋)
✅ 保持懷疑 (太聳動→可疑)
✅ 不隨意轉傳

【協作機制】
政府+媒體+平台+公民社會
共同建立:
├─ 快速澄清機制
├─ 教育宣導
├─ 透明化資訊來源
└─ 不侵害言論自由前提下打擊假訊息
```

**P2: Property (所有權)**

**定義**: **智慧財產權**,保護創作者對其創作的權利。

**三大智慧財產權** (109年普考考題!):

```
【1. 著作權 (Copyright)】

保護: 文學、藝術、音樂、軟體等創作
權利:
├─ 重製權 (複製)
├─ 公開傳輸權 (網路分享)
├─ 改作權
└─ 出租權

保護期限: 作者終身+死後50年

侵權範例:
❌ 盜版軟體
❌ 非法下載音樂、電影
❌ 網站盜用圖片
❌ 抄襲論文

合理使用 (Fair Use):
✅ 教學、研究用途
✅ 評論、新聞報導
✅ 少量引用 (需註明出處)

【2. 專利權 (Patent)】

保護: 發明、新型、設計
要件:
├─ 新穎性
├─ 進步性
└─ 產業利用性

保護期限: 20年 (發明)、10年 (新型)

範例:
├─ iPhone滑動解鎖專利
├─ 藥品專利
└─ 演算法專利 (爭議)

侵權範例:
❌ 未經授權製造專利產品

【3. 商標權 (Trademark)】

保護: 品牌名稱、Logo、標語
目的: 區別商品/服務來源

保護期限: 10年 (可續展)

範例:
├─ Apple蘋果Logo
├─ Nike勾勾
└─ Coca-Cola字體

侵權範例:
❌ 仿冒品牌
❌ 山寨Logo
```

**數位時代挑戰**:

```
【軟體著作權】
爭議:
├─ 開源 vs 商業軟體
├─ AI生成內容著作權歸誰?
└─ API是否受著作權保護?

【數位版權管理 (DRM)】
目的: 防止盜版
問題:
❌ 限制合法使用者權益
❌ 易被破解

【創用CC (Creative Commons)】
作者自主授權:
├─ CC0: 完全公有領域
├─ CC BY: 姓名標示
├─ CC BY-SA: 姓名標示-相同方式分享
└─ 平衡保護與分享
```

**A2: Accessibility (近用權)**

**定義**: 所有人**公平取得資訊的權利**,不因經濟、地理、身體等因素受限。

**核心議題**:

```
【數位落差 (Digital Divide)】

定義: 
不同群體在資訊近用上的差距

層次:
1. 第一層: 接近差距
   ├─ 有無網路、電腦
   └─ 城鄉差距

2. 第二層: 使用差距
   ├─ 會不會用
   └─ 年齡、教育差距

3. 第三層: 效益差距
   ├─ 用技術創造價值
   └─ 知識技能差距

影響因素:
├─ 經濟 (買不起設備)
├─ 地理 (偏鄉無網路)
├─ 年齡 (老年人不會用)
├─ 教育 (缺資訊素養)
└─ 身心障礙

【開放資料 (Open Data) - 105年高等二等考題!】

定義:
政府資料公開供民眾使用

台灣成就:
✅ 2016年全球排名第一!
  (英國開放知識基金會評比)

重要性:
├─ 政府透明化
├─ 促進創新 (開發App)
├─ 公民監督
└─ 經濟價值

範例:
├─ 政府資料開放平台 data.gov.tw
├─ 公共運輸資料 (開發查詢App)
├─ 空氣品質資料
└─ 實價登錄

挑戰:
❌ 機關主動性不足
❌ 資料品質不一
❌ 更新不及時

推動方式:
✅ 列為政府KPI
✅ 辦理黑客松 (Hackathon)
✅ 獎勵優秀應用
✅ 公私協力

【網站無障礙】
確保身心障礙者可使用網站:
├─ 螢幕報讀器相容
├─ 鍵盤操作
├─ 字體放大
└─ 高對比模式

【數位平權】
政府政策:
├─ 偏鄉網路建設
├─ 低收入戶上網補助
├─ 樂齡資訊教育
└─ 公共電腦據點
```

**PAPA模式總結**:

| 議題 | 核心問題 | 當代挑戰 | 對策 |
|------|---------|---------|------|
| **Privacy** | 誰可以使用我的資料? | 大數據、AI、監控 | 個資法、GDPR |
| **Accuracy** | 資訊是否正確? 誰負責? | 假新聞、Deepfake | 事實查核、媒體識讀 |
| **Property** | 誰擁有資訊? | 數位盜版、AI創作 | 著作權法、CC授權 |
| **Accessibility** | 誰可以取得資訊? | 數位落差 | 開放資料、數位平權 |

#### (3) COVID實聯制 vs 實名制 (111年關務考題!)

**背景**: COVID-19疫情期間,台灣採用「簡訊實聯制」協助疫調。

**實名制 vs 實聯制比較**:

```
【實名制 (Real-name System)】

定義:
登記**真實身分**,可直接識別個人

範例:
├─ 網咖實名制 (身分證登記)
├─ 手機門號實名制
└─ 疫情初期:手寫資料 (姓名、電話、地址)

特性:
✅ 可識別個人
❌ 隱私風險高
❌ 資料可能被濫用

【實聯制 (Contact Tracing System)】

定義:
登記**聯絡方式**,平時無法識別個人
僅在需要時(如疫調)才能聯絡

台灣「簡訊實聯制」:
1. 店家申請QR Code
2. 民眾掃碼 → 發送簡訊到1922
3. 電信業者加密儲存 (手機號碼+場所代碼+時間)
4. 平時政府**看不到**資料
5. 確診者足跡 → 政府調閱 → 發送警示簡訊

特性:
✅ 平時隱匿身分
✅ 需要時可聯絡
✅ 28天自動刪除

【比較表】

面向      | 手寫實名制        | 簡訊實聯制
----------|------------------|-------------
隱私保護   | ❌ 低 (完整個資)  | ✅ 高 (僅手機號碼)
資料保存   | 門口紙本 (易外洩) | **加密存電信商**
識別性     | 高 (直接看到姓名)  | 低 (無法直接識別)
使用便利性 | 低 (每次手寫)     | **高 (掃碼1秒)**
資料用途   | 易被濫用 (如推銷) | **專用於疫調**
保存期限   | 不確定            | **28天自動刪**
```

**為何實聯制較能保護個資?**

```
【技術設計】

1. 資料分散儲存
   ├─ 不是政府集中儲存
   └─ 分散在5家電信商

2. 加密保護
   └─ SHA-256單向雜湊

3. 最小化資料
   ├─ 僅儲存: 手機號碼+場所代碼+時間
   └─ 不知道: 姓名、身分證號、地址

4. 自動刪除
   └─ 28天後自動刪除

5. 用途限制
   ├─ 法律明定僅供疫調
   └─ 違法使用→刑責

6. 權限控管
   ├─ 需確診者足跡才能調閱
   ├─ 需多人授權
   └─ 有稽核紀錄

【vs 手寫紙本】

手寫紙本風險:
❌ 放在門口 → 任何人可偷看
❌ 完整個資 (姓名、電話、地址)
❌ 店家可能濫用 (推銷)
❌ 紙本遺失 → 個資外洩
❌保存期限不明

簡訊實聯制優勢:
✅ 資料加密儲存
✅ 僅有手機號碼 (無姓名等)
✅ 政府平時看不到
✅ 28天自動刪除
✅ 有法律保障

【實際案例】
疫情期間:
└─ 確診者到過餐廳A (5/1 12:00-13:00)
   └─ 政府向電信商調閱該時段、該場所的手機號碼
      └─ 發送簡訊: 「您可能與確診者有接觸,請自我健康監測」
         └─ 收到簡訊的人不知道誰確診 (隱私保護)

【爭議與挑戰】
爭議:
├─ 實聯制仍被警方違法調閱 (找人)
└─ 引發信任危機

挑戰:
├─ 確保僅用於疫調 (非治安)
├─ 疫情後是否廢除?
└─ 平衡防疫與隱私
```

#### (4) 外送平台策略分析 (114年高考新題!)

**情境**: 小餐館要不要加入外送平台 (如Uber Eats, Foodpanda)?

**分析角度**:

**1. 資訊系統策略角色**

```
【Porter策略角色】

支援性 (Support):
基本需要,無競爭優勢
→ 不加入 = 落後

工廠性 (Factory):
高依賴,但無差異化
→ 加入 = 不落後

轉型性 (Turnaround):
未來競爭優勢來源
→ 加入 = 搶先機

策略性 (Strategic):
當前競爭優勢
→ 加入 = 開拓新客源

【外送平台對餐館】
目前: 趨向**策略性/工廠性**
├─ 疫情後外送成主流
├─ 不加入 → 流失客戶
└─ 加入 → 擴大市場
```

**2. 交易成本理論**

```
交易成本 = 搜尋成本 + 協調成本 + 監督成本

【不加入】
客戶搜尋成本高:
├─ 客戶不知道餐館存在
├─ 難以比較菜單、評價
└─ 電話訂餐麻煩

協調成本高:
├─ 電話溝通訂單
├─ 地址錯誤
└─ 送達時間不確定

監督成本高:
├─ 難以追蹤外送員
└─ 客訴處理成本

【加入平台】
✅ 搜尋成本↓: 客戶在App一次看完
✅ 協調成本↓: 系統自動化
✅ 監督成本↓: 平台追蹤、評價機制

雖然要付平台費
但**降低的交易成本 > 平台費**
```

**3. 網路效應**

```
【雙邊市場網路效應】

平台價值 = f(餐廳數量 × 客戶數量)

正向循環:
更多餐廳 → 吸引更多客戶
    ↓           ↓
更多客戶 → 吸引更多餐廳

【對小餐館】
✅ 搭便車 (Free Ride)
   借用平台的大量客戶
   
✅ 突破地理限制
   原本僅附近居民
   → 全市都可能訂購

✅ 曝光機會
   平台推薦、搜尋排名
   
⚠️ 風險: 依賴平台
   平台抽成調漲 → 被動接受
```

**4. 資料導向決策**

```
【平台提供數據】

銷售數據:
├─ 哪道菜最熱銷?
├─ 尖峰時段?
├─ 客單價分析
└─ 季節趨勢

客戶回饋:
├─ 評價分析
├─ 常見抱怨
└─ 改善建議

競爭分析:
├─ 同類餐廳定價
├─ 促銷策略
└─ 市場空缺

【資料應用】

基於數據決策:
✅ 調整菜單 (下架冷門菜)
✅ 優化定價
✅ 安排備料 (預測需求)
✅ 推出限時優惠 (尖峰/離峰)

vs 無數據:
❌ 憑感覺決策
❌ 不知道客戶偏好
❌ 浪費備料
```

**效益總結**:

```
【有形效益】
✅ 營收增加 (新客源)
   小餐館: 外送可能佔營收30-50%
   
✅ 提高稼動率
   離峰時段仍有訂單

✅ 品牌曝光
   更多人認識餐館

【無形效益】
✅ 客戶數據洞察
✅ 拓展客群 (年輕人)
✅ 競爭力提升

【成本】
❌ 平台抽成 (25-35%)
❌ 行銷費用
❌ 外送包裝成本

【建議】
視為**獲客管道**
目標: 平台客戶 → 轉為自有客戶
(如優惠券吸引直接電話訂購)
```
