## 📚 【核心知識】系統安全完全解析

### 一、OWASP Top 10 常見漏洞

1.  **Injection (注入攻擊)**：
    *   **SQL Injection**：輸入 SQL 指令操作資料庫。
    *   **Command Injection**：輸入 OS 指令控制主機。
2.  **XSS (Cross-Site Scripting)**：
    *   在網頁植入惡意 Script，竊取使用者 Cookie 或 Session。
3.  **Broken Access Control (存取控制失效)**：
    *   未驗證權限，導致越權存取他人資料 (IDOR)。

### 二、惡意程式 (Malware) 類型

*   **Virus (病毒)**：需寄生於檔案，需人為執行才能傳播。
*   **Worm (蠕蟲)**：獨立程式，利用網路漏洞自動傳播 (如 WannaCry)。
*   **Trojan (木馬)**：偽裝成正常軟體，開啟後門。
*   **Ransomware (勒索軟體)**：加密檔案勒索贖金。

### 三、端點防護演進

*   **Antivirus (防毒)**：特徵碼比對，只能擋已知病毒。
*   **EDR (Endpoint Detection and Response)**：行為分析，可擋未知威脅，具備蒐證與回應能力。
*   **MDR (Managed Detection and Response)**：EDR 工具 + 真人專家服務 (SOC)。

### 【題型一】端點偵測與回應 (EDR) 與 MDR

#### 📖 原題 (113年高考三級)

> **題目**：
> 1. 何謂端點偵測與回應（EDR）及託管偵測與回應（MDR）機制。
> 2. 請分別說明 EDR 及 MDR 在偵測方面及回應方面有那些活動。

#### 🎯 答題架構分析

1.  **定義**：EDR (工具) vs MDR (服務)。
2.  **活動比較**：列表比較兩者在偵測 (行為 vs 獵捕) 與回應 (隔離 vs 調查) 的差異。

#### ✍️ 標準答案示範

**解答**：

**1. 定義說明**
*   **EDR (Endpoint Detection and Response)**：安裝在端點 (PC, Server) 上的軟體工具，負責持續監控與蒐集端點行為數據，利用大數據分析技術偵測可疑活動，並提供自動或手動的回應功能。
*   **MDR (Managed Detection and Response)**：一種 **資安委外服務**。由外部專業資安團隊 (SOC) 運用 EDR 等工具，為客戶提供 24/7 的威脅監控、分析與事件處理服務。簡單說：EDR 是工具，MDR 是「工具 + 專家服務」。

**2. 偵測與回應活動比較**

| 構面 | EDR (工具功能) | MDR (專家服務) |
| :--- | :--- | :--- |
| **偵測 (Detection)** | 1. **行為監控**：記錄 Process, Registry, File 異動。<br>2. **威脅情資比對**：比對 IOC (Hash, IP)。<br>3. **異常分析**：利用 ML 偵測無檔案攻擊 (Fileless)。 | 1. **進階威脅獵捕 (Threat Hunting)**：專家主動挖掘潛伏威脅。<br>2. **警報過濾**：人工分析 EDR 警報，排除誤報 (False Positive)。<br>3. **關聯分析**：跨端點、跨網路分析攻擊路徑。 |
| **回應 (Response)** | 1. **隔離主機**：切斷網路連線。<br>2. **刪除程序**：終止惡意 Process。<br>3. **還原檔案**：透過 Shadow Copy 還原被加密檔案。 | 1. **事件調查**：分析攻擊根因 (Root Cause)。<br>2. **應變指導**：提供修補建議與資安強化策略。<br>3. **客製化處理**：針對特定攻擊手法調整防禦規則。 |

**評分標準** (預估)：
*   定義清楚：8分
*   偵測活動比較：8分
*   回應活動比較：9分

### 【題型二】社交工程與技術性攻擊

#### 📖 原題 (114年高考三級)

> **題目**：請說明社交工程攻擊與技術性攻擊（如惡意程式、SQL Injection）在手法與防禦策略上的差異，並分析企業若僅依賴技術防禦可能面臨的風險。

#### 🎯 答題架構分析

1.  **差異比較**：目標 (人 vs 系統)、手法、防禦。
2.  **風險分析**：人是漏洞、繞過邊界、BEC 詐騙。

#### ✍️ 標準答案示範

**解答**：

**1. 差異比較**

| 項目 | 社交工程 (Social Engineering) | 技術性攻擊 (Technical Attack) |
| :--- | :--- | :--- |
| **攻擊目標** | **人 (People)** 的心理弱點 (信任、恐懼、貪婪)。 | **系統 (System)** 的技術漏洞 (Code, Config)。 |
| **常見手法** | 釣魚郵件 (Phishing)、假冒身分電話 (Vishing)、USB 丟棄攻擊。 | SQL Injection, Buffer Overflow, Exploit Kit。 |
| **防禦策略** | **教育訓練**、社交工程演練、雙重驗證 (MFA)。 | **修補漏洞 (Patch)**、防火牆、WAF、防毒軟體。 |

**2. 僅依賴技術防禦的風險**
*   **人是最大的漏洞**：即使防火牆再強，若員工被騙主動交出帳號密碼或執行惡意附檔，技術防禦將形同虛設。
*   **繞過邊界防護**：社交工程常透過合法管道 (如 Email) 進入，且使用合法帳號權限，傳統資安設備難以偵測異常。
*   **BEC (商務電子郵件詐騙)**：這類攻擊通常不含惡意程式，純粹利用話術詐騙匯款，技術偵測極難防範。

**評分標準** (預估)：
*   差異比較表 (3項)：15分
*   風險分析 (3點)：10分

### 【題型三】SQL Injection 攻擊與防禦

#### 📖 原題 (113年普考)

> **題目**：針對 SQL 注入攻擊（SQL Injection），請回答下列問題：
> 1. 請說明何謂 SQL 注入攻擊；並說明此攻擊的常見實務案例。
> 2. 請說明使用預備語句（Prepared Statements）可以有效防止 SQL 注入攻擊的理由。

#### 🎯 答題架構分析

1.  **定義與案例**：說明原理，舉登入繞過 (`' OR '1'='1`) 為例。
2.  **防禦原理**：說明 Prepared Statement 如何將結構與資料分離。

#### ✍️ 標準答案示範

**解答**：

**1. SQL Injection 定義與案例**
*   **定義**：攻擊者將惡意 SQL 指令插入到應用程式的輸入欄位中，欺騙後端資料庫執行該指令，藉此竊取資料、繞過驗證或破壞資料庫。
*   **實務案例 (繞過登入)**：
    *   正常 SQL：`SELECT * FROM users WHERE username = '$user' AND password = '$pwd'`
    *   攻擊輸入：Username 輸入 `' OR '1'='1`
    *   結果 SQL：`SELECT * FROM users WHERE username = '' OR '1'='1' AND password = '...'`
    *   **後果**：因為 `'1'='1'` 永遠為真 (True)，攻擊者無需密碼即可成功登入。

**2. Prepared Statements 防禦原理**
*   **原理**：將 SQL 語法結構與資料參數分開處理。
*   **過程**：
    1.  **Prepare**：資料庫先編譯 SQL 模板 (如 `SELECT * FROM users WHERE username = ?`)，鎖定語法結構。
    2.  **Bind**：將使用者輸入的資料填入 `?` 的位置。
*   **理由**：因為 SQL 結構已經在第一步鎖定，使用者輸入的任何內容 (包含 `' OR '1'='1`) 都只會被視為單純的「字串資料」，而不會被當作 SQL 指令執行，徹底根除注入風險。

**評分標準** (預估)：
*   定義與案例 (含 SQL 範例)：15分
*   防禦原理 (結構與資料分離)：10分
