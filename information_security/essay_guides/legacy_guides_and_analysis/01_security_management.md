## 📚 【核心知識】資安管理制度完全解析

### 一、ISO 27001 資訊安全管理系統 (ISMS)

**定義**：
一套國際通用的標準，提供系統化的方法來管理敏感資訊，確保其**機密性 (Confidentiality)**、**完整性 (Integrity)** 及 **可用性 (Availability)**。

**核心精神：PDCA 循環**
*   **Plan (規劃)**：建立 ISMS 政策、目標、進行風險評鑑。
*   **Do (執行)**：實施控制措施、運作 ISMS。
*   **Check (查核)**：內部稽核、管理審查、績效量測。
*   **Act (行動)**：矯正措施、持續改善。

### 二、風險管理 (Risk Management)

**風險公式**：
$$風險 (Risk) = 資產價值 (Asset) \times 威脅 (Threat) \times 弱點 (Vulnerability)$$

**風險評鑑三階段**：
1.  **風險識別**：找出資產、威脅與弱點。
2.  **風險分析**：推估發生的可能性與衝擊，計算風險值。
3.  **風險評估**：將風險值與可接受水準比較，決定優先順序。

**風險處理四策略 (T.A.M.A.)**：
1.  **Transfer (移轉)**：買保險、委外。
2.  **Avoid (規避)**：停止該業務。
3.  **Mitigate (降低)**：實施控制措施 (如防火牆)。
4.  **Accept (接受)**：風險低，持續監控。

### 【題型一】ISO 27001 標準與效益

#### 📖 原題 (112年高考三級)

> **題目**：
> 1. 請說明 ISO 27001 資訊安全管理系統，並列舉出相關內外部風險。
> 2. 請列舉五項通過 ISO 27001 資訊安全管理系統驗證的效益。

#### 🎯 答題架構分析

1.  **ISO 27001 定義**：先破題，說明這是什麼標準 (ISMS, CIA, PDCA)。
2.  **內外部風險**：依據 ISO 27001 第 4 章「組織全景」，分別列舉內部與外部議題。
3.  **驗證效益**：從法規、信任、風險、文化、營運等不同面向列舉 5 點。

#### ✍️ 標準答案示範

**問題 (1)：ISO 27001 說明與內外部風險**

**解答**：

**1. ISO 27001 資訊安全管理系統說明**
ISO/IEC 27001 是國際通用的資訊安全管理系統 (ISMS) 標準。它採用 **PDCA (Plan-Do-Check-Act)** 的循環管理模式，旨在協助組織建立、實作、維護並持續改善資訊安全，以確保資訊資產的 **機密性 (Confidentiality)**、**完整性 (Integrity)** 與 **可用性 (Availability)**。

**2. 相關內外部風險 (依據組織全景)**
依據標準要求，組織需考量可能影響 ISMS 預期成果的內外部議題：

*   **內部風險 (Internal Issues)**：
    *   **人員意識不足**：員工缺乏資安認知，誤點釣魚郵件或設定弱密碼。
    *   **資源分配限制**：資安預算或人力不足，無法及時更新防護設備。
    *   **流程與治理缺陷**：權限控管不當，或缺乏明確的資安政策。
    *   **老舊系統風險**：使用已停止維護 (EOL) 的軟硬體，存在已知漏洞。

*   **外部風險 (External Issues)**：
    *   **網路威脅演變**：駭客攻擊手法翻新 (如勒索軟體、APT 攻擊)。
    *   **法規遵循要求**：資通安全管理法、個資法等法規修訂，增加合規壓力。
    *   **供應鏈風險**：委外廠商或合作夥伴發生資安事故，連帶波及本組織。
    *   **環境與天災**：地震、火災或電力中斷等不可抗力因素。

**問題 (2)：通過驗證之五項效益**

**解答**：

1.  **符合法規與合規要求**：滿足《資通安全管理法》對特定等級機關需導入 ISMS 的要求，避免法律風險。
2.  **提升客戶與利害關係人信任**：透過第三方公正驗證，向客戶證明組織具備國際水準的資安防護能力，增加商業競爭力。
3.  **有效降低資安風險**：透過系統化的風險評鑑與處理機制，能主動識別並降低資安事件發生的機率與衝擊。
4.  **建立全員資安文化**：透過標準要求的教育訓練與全員參與，提升整體員工的資安意識，減少人為疏失。
5.  **確保營運持續 (BCP)**：ISMS 強調營運持續管理的整合，確保在災害發生時能快速恢復核心業務運作，減少損失。

**評分標準** (預估)：
*   ISO 27001 定義清楚 (CIA, PDCA)：5分
*   內外部風險各列舉 2-3 項且合理：5分
*   效益列舉 5 項且涵蓋不同面向：5分

### 【題型二】資安事件應變 (IR)

#### 📖 原題 (114年高考三級)

> **題目**：企業在面對重大資安事件（如勒索病毒入侵或大規模資料外洩）時，若無有效的資安事件應變計畫，可能產生那些影響？請舉例說明，並提出具體改善建議。

#### 🎯 答題架構分析

1.  **負面影響**：從擴散、決策、商譽、法規四個角度切入。
2.  **改善建議**：對應上述影響，提出 CSIRT、SOP、演練、備份等具體作法。

#### ✍️ 標準答案示範

**解答**：

**1. 缺乏有效應變計畫之影響**

若缺乏完善的資安事件應變計畫 (Incident Response Plan, IRP)，在面對勒索病毒或資料外洩時，可能造成以下嚴重後果：

1.  **損害範圍擴大 (擴散風險)**：
    *   *說明*：無法在第一時間隔離受駭主機，導致勒索病毒透過內網橫向擴散，癱瘓全公司系統。
2.  **錯失黃金處置時間 (決策癱瘓)**：
    *   *說明*：缺乏明確的指揮體系與通報流程，員工不知所措，延誤止血時機，導致資料外洩量增加。
3.  **商譽與信任崩盤 (公關危機)**：
    *   *說明*：對外發言不一致或試圖隱瞞，遭媒體披露後引發大眾恐慌與客戶不信任。
4.  **法律合規風險 (鉅額罰款)**：
    *   *說明*：未依《資通安全管理法》或《個資法》規定在時限內完成通報，面臨主管機關裁罰。

**2. 具體改善建議**

為強化資安韌性，建議採取以下措施：

1.  **建立 CSIRT 應變組織**：
    *   明確定義事件指揮官 (Commander)、發言人、技術檢測人員的職責 (Roles & Responsibilities)，確保指揮體系順暢。
2.  **制定標準作業程序 (SOP) 與 Playbook**：
    *   針對高風險威脅 (如勒索軟體、DDoS、網頁置換) 制定專屬的應變劇本，詳列偵測、隔離、根除、復原的具體步驟。
3.  **定期辦理攻防演練 (Drill)**：
    *   每年至少進行一次社交工程演練與資安事件通報演練，模擬真實攻擊情境，驗證 SOP 的有效性並修正缺失。
4.  **落實備份 3-2-1 原則與離線備份**：
    *   確保擁有 **離線 (Offline)** 或 **不可變更 (Immutable)** 的備份資料，即使線上系統全毀，仍能從乾淨備份快速復原，不需支付贖金。
5.  **建立聯防機制**：
    *   加入 TWCERT/CC 或相關 ISAC (資安資訊分享與分析中心)，取得外部情資與技術支援。

**評分標準** (預估)：
*   影響列舉 4 點且具體：12分
*   改善建議列舉 4-5 點且具體：13分

### 【題型三】資安風險評估步驟

#### 📖 原題 (110年特考三級)

> **題目**：請回答下列資訊系統風險評鑑相關問題：
> 1. 請說明風險識別及其步驟。
> 2. 請說明風險估計及其步驟。
> 3. 請說明風險評估。

#### 🎯 答題架構分析

這題考的是 **NIST SP 800-30** 或 **ISO 27005** 的標準流程。必須清晰區分「識別」、「估計(分析)」與「評估(評量)」三個階段的定義與細部步驟。

#### ✍️ 標準答案示範

**解答**：

風險評鑑 (Risk Assessment) 是風險管理的核心過程，依序包含以下三個階段：

**1. 風險識別 (Risk Identification)**
*   **說明**：此階段旨在找出可能影響組織資產機密性、完整性或可用性的所有潛在風險來源。
*   **步驟**：
    1.  **資產盤點 (Asset Inventory)**：識別並分類所有的資訊資產 (如伺服器、資料庫、人員、文件)。
    2.  **識別威脅 (Threat Identification)**：找出可能危害資產的潛在來源 (如駭客攻擊、惡意員工、天災)。
    3.  **識別弱點 (Vulnerability Identification)**：找出資產本身或管控措施的脆弱點 (如未修補的漏洞、弱密碼)。
    4.  **識別現有控制措施**：確認目前已實施的防護機制 (如已安裝防火牆)。

**2. 風險估計/分析 (Risk Estimation/Analysis)**
*   **說明**：針對已識別的風險，分析其發生的可能性與造成的衝擊，以量化或質化方式計算風險值。
*   **步驟**：
    1.  **分析可能性 (Likelihood)**：評估威脅利用弱點導致事故發生的機率 (例如：高/中/低，或 1-5 分)。
    2.  **分析衝擊 (Impact)**：評估事故發生後對組織造成的損害程度 (如財務損失、商譽受損、營運中斷)。
    3.  **計算風險值 (Risk Level)**：依據公式計算風險等級。
        $$風險值 = 可能性 \times 衝擊$$

**3. 風險評估/評量 (Risk Evaluation)**
*   **說明**：將計算出的風險值與組織的風險準則進行比較，以決定風險處理的優先順序。
*   **內容**：
    *   設定 **風險胃納 (Risk Appetite)** 或 **可接受風險水準**。
    *   將風險值高於可接受水準的項目列為優先處理對象。
    *   產出 **風險處理計畫 (Risk Treatment Plan)**，決定採取降低、移轉、規避或接受等策略。

**評分標準** (預估)：
*   風險識別定義與 3-4 個步驟：8分
*   風險估計定義與 3 個步驟 (可能性/衝擊/風險值)：8分
*   風險評估定義與決策過程：4分

### 【題型四】ISO 27001 PDCA 循環實務

#### 📖 原題 (108年高考二級 / 105年高考三級)

> **題目**：請說明導入資訊安全管理系統 (ISMS) 的四個階段 (Plan, Do, Check, Act) 之主要工作項目。

#### 🎯 答題架構分析

這是資安管理最基礎也最重要的題目。必須清楚列出 PDCA 四個階段的**定義**以及**具體產出** (如 P 階段產出 SOA，D 階段產出紀錄)。

#### ✍️ 標準答案示範

**解答**：

ISO 27001 採用 PDCA (Plan-Do-Check-Act) 循環作為持續改善的基礎，各階段主要工作如下：

**1. 規劃 (Plan)：建立 ISMS**
*   **目標**：依據組織的風險胃納與資安目標，建立相關政策、目標、流程與程序。
*   **主要工作**：
    *   **界定範圍 (Scope)**：確認 ISMS 涵蓋的組織單位、實體位置與資產。
    *   **風險評鑑 (Risk Assessment)**：識別資產、威脅與弱點，並評估風險等級。
    *   **產出適用性聲明書 (SOA)**：決定要採用哪些控制措施 (Controls) 來處理風險。

**2. 執行 (Do)：實作與運作 ISMS**
*   **目標**：實作規劃階段所制定的政策、控制措施與程序。
*   **主要工作**：
    *   **實施風險處理計畫**：部署防火牆、加密、備份等技術與管理措施。
    *   **教育訓練**：提升人員資安意識與專業技能。
    *   **維運管理**：執行日常資安作業 (如 Log 審查、弱點掃描) 並留存紀錄。

**3. 查核 (Check)：監控與審查 ISMS**
*   **目標**：評估 ISMS 的執行成效是否符合預期目標與政策要求。
*   **主要工作**：
    *   **績效量測**：監控資安指標 (KPI)，如事件發生率、修補時效。
    *   **內部稽核 (Internal Audit)**：定期檢查 ISMS 是否符合標準要求。
    *   **管理審查 (Management Review)**：高階主管定期審查 ISMS 的適用性與有效性。

**4. 行動 (Act)：維護與改善 ISMS**
*   **目標**：針對查核階段發現的缺失進行矯正，確保持續改善。
*   **主要工作**：
    *   **矯正措施 (Corrective Action)**：針對不符合事項 (Non-conformity) 進行根因分析並修正。
    *   **持續改善**：依據內外部環境變遷，調整資安策略，提升整體防護水準。

**評分標準** (預估)：
*   四階段定義正確：10分
*   各階段主要工作項目具體：15分

### 【題型五】風險處理策略與勒索軟體

#### 📖 原題 (109年特考三級)

> **題目**：組織遭受勒索攻擊的風險日益升高，請說明風險處理 (Risk Treatment) 的四種策略，並針對勒索軟體攻擊提出降低風險之具體措施。

#### 🎯 答題架構分析

1.  **四種策略**：TAMA (移轉、規避、降低、接受)。
2.  **勒索軟體對策**：針對「降低」策略，提出備份、修補、教育訓練等作法。

#### ✍️ 標準答案示範

**解答**：

**1. 風險處理四種策略**
針對評鑑出的風險，組織可採取以下四種處理策略 (T.A.M.A.)：
1.  **降低 (Mitigate / Modify)**：採取控制措施來降低風險發生的可能性或衝擊。
    *   *例*：安裝防火牆、定期修補漏洞。
2.  **移轉 (Transfer / Share)**：將風險轉嫁給第三方承擔。
    *   *例*：購買資安保險、將機房維運委外。
3.  **規避 (Avoid)**：停止產生風險的活動或業務，徹底消除風險。
    *   *例*：關閉有嚴重漏洞且無法修補的舊系統。
4.  **接受 (Accept / Retain)**：當風險低於可接受水準，或處理成本高於資產價值時，選擇維持現狀並持續監控。
    *   *例*：接受測試環境的低風險漏洞，暫不修補。

**2. 針對勒索軟體之降低風險措施**
針對勒索軟體威脅，應優先採取「降低」策略，具體措施如下：
1.  **落實備份 3-2-1 原則**：至少 3 份備份、2 種不同媒體、1 份異地 (且最好離線)，確保資料可回復。
2.  **定期修補漏洞**：勒索軟體常利用 OS 或軟體漏洞 (如 SMB) 進行滲透，應即時更新 Patch。
3.  **強化電子郵件防護**：部署郵件過濾系統攔截釣魚信件，並加強員工不開啟不明附檔的資安意識。
4.  **限制權限與網段隔離**：落實最小權限原則，並將內網區隔，避免單一主機中毒後橫向擴散至全網。

**評分標準** (預估)：
*   四種策略定義與舉例：12分
*   勒索軟體防護措施：13分

### 【題型六】資安稽核 (Internal vs External Audit)

#### 📖 原題 (107年普考)

> **題目**：請說明資安稽核 (Audit) 之目的，以及內部稽核與外部稽核之差異。

#### 🎯 答題架構分析

1.  **目的**：合規、有效性、發現缺失。
2.  **差異比較**：執行者、目的、頻率、結果效力。

#### ✍️ 標準答案示範

**解答**：

**1. 資安稽核之目的**
資安稽核是透過系統化、獨立且文件化的過程，獲取稽核證據並客觀評估，以達成以下目的：
1.  **驗證合規性**：確認組織是否符合法規 (如資安法)、標準 (如 ISO 27001) 或內部政策的要求。
2.  **評估有效性**：檢查現有的控制措施是否有效運作，能否達成預期的資安目標。
3.  **發現缺失與改善**：找出潛在的資安漏洞或管理盲點，提供改善建議以強化防護。

**2. 內部稽核與外部稽核之差異**

| 比較項目 | 內部稽核 (Internal Audit) | 外部稽核 (External Audit) |
| :--- | :--- | :--- |
| **執行者** | 組織內部受過訓練的員工，或委託外部顧問 (但代表內部)。 | 獨立的第三方驗證機構 (如 BSI, SGS) 之稽核員。 |
| **主要目的** | 自我檢查、管理審查輸入、準備外部稽核。 | 取得公正的驗證證書、向利害關係人證明合規。 |
| **頻率** | 通常較頻繁 (如每半年或一年一次)。 | 通常一年一次 (複評) 或三年一次 (重評)。 |
| **結果效力** | 內部改善報告，供管理階層決策參考。 | 具公信力的稽核報告與證書，具法律或商業效力。 |
| **關注點** | 重點在於流程的持續改善與落實。 | 重點在於是否符合標準條文的最低要求。 |

**評分標準** (預估)：
*   稽核目的 (3點)：10分
*   內外稽核比較表 (4-5項)：15分
