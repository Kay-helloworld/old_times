## 📚 【核心知識】網路安全完全解析

### 一、防火牆演進 (Firewall Evolution)

1.  **封包過濾 (Packet Filtering)**：L3/L4，檢查 IP/Port。
2.  **狀態檢測 (Stateful Inspection)**：L4，追蹤連線狀態 (ESTABLISHED)。
3.  **應用層閘道 (Proxy)**：L7，代理連線，效能較差。
4.  **WAF (Web Application Firewall)**：L7，專擋 HTTP 攻擊 (SQLi, XSS)。
5.  **NGFW (Next-Generation Firewall)**：整合 IPS、App-ID、User-ID、沙箱。

### 二、VPN 技術 (Virtual Private Network)

*   **IPSec VPN**：
    *   運作於 L3 (Network Layer)。
    *   需安裝 Client 軟體，適合 Site-to-Site 連線。
    *   安全性高，設定複雜。
*   **SSL/TLS VPN**：
    *   運作於 L4/L7 (Transport/App Layer)。
    *   透過瀏覽器即可連線 (Clientless)，適合 Client-to-Site (遠距辦公)。
    *   便利性高，細粒度存取控制。

### 三、零信任架構 (Zero Trust Architecture, ZTA)

**核心精神**：**永不信任，始終驗證 (Never Trust, Always Verify)**。
不再以內外網區分信任邊界，所有存取請求 (無論來自內部或外部) 都必須經過身分驗證、授權與加密。

### 【題型一】防火牆技術：WAF 與 NGFW

#### 📖 原題 (113年高考三級)

> **題目**：防火牆用以保障內部網路避免受攻擊，目前常被應用的有 WAF（Web Application Firewall）及次世代防火牆（Next-Generation Firewall, NGFW），試問：
> 1. WAF 的防禦機制為何？
> 2. 次世代防火牆的防禦機制為何？
> 3. 當內容傳遞網路 CDN 與 WAF 架設在一起時，其效益為何？

#### 🎯 答題架構分析

1.  **WAF 機制**：鎖定 L7 HTTP/S，特徵碼、行為分析、OWASP Top 10。
2.  **NGFW 機制**：App-ID, User-ID, IPS, Sandbox。
3.  **CDN+WAF 效益**：效能(快取)、防護(DDoS)、隱匿(源站IP)。

#### ✍️ 標準答案示範

**解答**：

**1. WAF (Web Application Firewall) 防禦機制**
WAF 專門針對 OSI 模型第 7 層 (應用層) 的 HTTP/HTTPS 流量進行深層檢測，其核心機制包括：
*   **特徵碼比對 (Signature-based)**：比對已知攻擊模式，阻擋 SQL Injection、XSS 等常見 Web 攻擊。
*   **行為分析 (Anomaly Detection)**：建立正常流量基線，偵測異常請求 (如異常的 Payload 長度)。
*   **虛擬修補 (Virtual Patching)**：在應用程式原始碼修補前，先透過 WAF 規則阻擋針對特定漏洞的攻擊。

**2. NGFW (Next-Generation Firewall) 防禦機制**
NGFW 突破傳統防火牆僅能依據 IP/Port 過濾的限制，具備以下進階機制：
*   **應用程式識別 (App-ID)**：能識別封包內的應用程式 (如 Facebook, Line)，而非僅看 Port 80/443。
*   **使用者識別 (User-ID)**：結合 AD/LDAP，依據使用者身分與群組進行存取控制。
*   **整合入侵防禦 (IPS)**：內建深度封包檢測 (DPI)，即時阻擋已知漏洞攻擊。
*   **沙箱檢測 (Sandboxing)**：將未知檔案送至雲端沙箱模擬執行，偵測惡意軟體。

**3. CDN 與 WAF 整合之效益**
*   **強化 DDoS 防護**：CDN 的分散式節點可吸收巨量流量攻擊 (Volumetric DDoS)，WAF 則負責過濾應用層攻擊 (Layer 7 DDoS)，形成多層次防禦。
*   **提升存取效能**：CDN 快取靜態內容，降低 WAF 與源站伺服器的負載，加速網頁載入。
*   **隱藏源站 IP**：攻擊者僅能看到 CDN 節點 IP，無法直接攻擊源站，大幅提升安全性。

**評分標準** (預估)：
*   WAF 機制 (L7, OWASP)：8分
*   NGFW 機制 (App-ID, IPS)：8分
*   CDN 效益 (DDoS, 效能)：9分

### 【題型二】分散式阻斷服務 (DDoS)

#### 📖 原題 (112年高考三級)

> **題目**：
> 1. 請說明分散式阻斷服務的原理與難以防守之原因。
> 2. 請列舉五項分散式阻斷服務的防護策略。

#### 🎯 答題架構分析

1.  **原理**：Botnet, 資源耗盡。
2.  **難防守原因**：分散源頭、偽造 IP、模擬正常流量。
3.  **防護策略**：清洗中心、CDN、Rate Limiting、Geo-blocking、系統優化。

#### ✍️ 標準答案示範

**解答**：

**1. DDoS 原理與難以防守原因**
*   **原理**：攻擊者控制大量遭駭的殭屍電腦 (Botnet)，同時向目標發送巨量請求，耗盡目標的網路頻寬、系統資源 (CPU/RAM) 或連線數，導致正常使用者無法存取服務。
*   **難以防守原因**：
    1.  **攻擊源分散**：流量來自全球各地，難以單靠封鎖特定 IP 解決。
    2.  **偽造來源 IP (IP Spoofing)**：攻擊封包常偽造來源，難以追蹤真實攻擊者。
    3.  **模擬正常流量**：應用層攻擊 (如 HTTP Flood) 模擬正常瀏覽行為，傳統防火牆難以區分。
    4.  **反射放大攻擊**：利用 DNS、NTP 等協議放大攻擊流量，攻擊成本低但威力巨大。

**2. 五項防護策略**
1.  **流量清洗服務 (Scrubbing Center)**：與 ISP 合作，將攻擊流量引導至清洗中心進行過濾，只將乾淨流量導回企業網路。
2.  **採用 CDN 服務**：利用 CDN 龐大的頻寬與分散式架構吸收攻擊流量，並隱藏源站真實 IP。
3.  **防火牆/IPS 速率限制 (Rate Limiting)**：設定單一 IP 或網段的連線數與請求頻率上限，防止資源被單一來源耗盡。
4.  **地理位置封鎖 (Geo-blocking)**：若業務僅限國內，可封鎖來自高風險國家或地區的 IP 流量。
5.  **系統參數優化**：關閉不必要的服務 (如 UDP Port)，調整 TCP Timeout 設定 (如縮短 SYN-Received 等待時間)，增加系統抗打擊能力。

**評分標準** (預估)：
*   原理與難防守原因 (4點)：12分
*   防護策略 (5點)：13分

### 【題型三】VPN 的優勢與風險

#### 📖 原題 (109年高考二級 / 114年普考)

> **題目**：因疫情關係，許多組織企業鼓勵員工在家上班，利用 VPN 連回公司處理公務。請說明 VPN 的優勢與可能面臨的資安風險。

#### 🎯 答題架構分析

1.  **優勢**：CIA (機密性/完整性)、成本、存取控制。
2.  **風險**：端點淪陷、跳板攻擊、分割通道。
3.  **防護**：MFA、NAC、Full Tunneling。

#### ✍️ 標準答案示範

**解答**：

**1. VPN 優勢**
1.  **機密性 (Confidentiality)**：透過加密隧道 (Tunneling) 傳輸資料，防止在公共網路 (Internet) 上被竊聽。
2.  **完整性 (Integrity)**：確保資料在傳輸過程中未被竄改。
3.  **存取控制**：員工可透過 VPN 取得內網 IP，存取僅限內網使用的資源 (如檔案伺服器、ERP)，實現邊界延伸。
4.  **成本效益**：利用公共網路建立虛擬私有網路，比租用實體專線 (Leased Line) 便宜許多。

**2. 資安風險與防護**
1.  **端點淪陷 (Compromised Endpoint)**：若員工家中電腦中毒，VPN 連線後病毒可能透過隧道感染公司內網。
    *   *防護*：實施 **NAC (網路存取控制)**，檢查端點健康狀態 (如防毒更新) 才允許連線。
2.  **帳號密碼遭竊**：駭客竊取 VPN 帳密後可長驅直入。
    *   *防護*：強制啟用 **MFA (多因子驗證)**。
3.  **分割通道 (Split Tunneling) 風險**：若允許員工同時連 VPN 和一般 Internet，駭客可能從 Internet 入侵員工電腦再跳板進內網。
    *   *防護*：強制全通道 (Full Tunneling) 或嚴格限制分割通道策略。

**評分標準** (預估)：
*   優勢 (4點)：12分
*   風險與防護 (3點)：13分
