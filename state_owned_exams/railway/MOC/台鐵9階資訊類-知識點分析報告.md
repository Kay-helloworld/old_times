## 🎯 兩週衝刺策略建議

### 優先級矩陣

| 優先度 | 知識點 | 理由 |
|--------|--------|------|
| 🔴 P0 必讀 | 風險管理（PMBOK風險流程）| 科目三佔40%題數，幾乎全背就高分 |
| 🔴 P0 必讀 | PMBOK五大過程群組 | 科目三基礎，科目二也出現 |
| 🔴 P0 必讀 | OSI各層功能 + 各層設備 | 科目一必考，難度低易拿分 |
| 🔴 P0 必讀 | 應用層常見協定 | SMTP/HTTP/DNS/Telnet，背定義就拿分 |
| 🟠 P1 應讀 | 加密技術（對稱/非對稱）| 科目一資安必考 |
| 🟠 P1 應讀 | 資料庫正規化 + ER圖 | 科目二常出現 |
| 🟠 P1 應讀 | 軟體測試（白箱/黑箱）| 科目二常考 |
| 🟠 P1 應讀 | LAN/WLAN技術（IEEE 802）| 科目一5題連出 |
| 🟡 P2 補充 | 訊號與編碼（NRZ/Manchester）| 科目一有6題但難度高，性價比低 |
| 🟡 P2 補充 | EVM實獲值計算 | 計算題，需練習 |
| 🟢 P3 餘力 | WAN技術（X.25/InfiniBand）| 題數少且偏門 |
| 🟢 P3 餘力 | PKI/CA詳細機制 | 只考1題 |

### 時間分配建議（2週）

```
第1週（6天）：
Day1-2：科目三 - PMBOK框架 + 風險管理（背最多，性價比最高）
Day3-4：科目一 - OSI模型 + 各層設備 + 應用層協定
Day5-6：科目一 - 網路技術（LAN/IP/TCP/路由）

第2週（6天）：
Day1-2：科目二 - 資料庫設計 + 軟體設計（SDLC/OOP/測試）
Day3-4：科目一 - 資通安全（加密 + 存取控制 + 攻擊手法）
Day5-6：科目三 - 時程/成本/品質管理 + 全科弱點補強
```
