# CPU指令週期

## 核心定義

> 一句話定義：CPU 從主記憶體中取出指令、解說指令意義、到實際執行並儲存結果的完整循環過程。

## 詳細說明

CPU 在執行每一道機器指令時，都會遵循一個固定的基本步驟迴圈，稱為「指令週期」。
標準的四個基本步驟為（**非常愛考排序**）：
1. **擷取 (Fetch)**：從主記憶體 (RAM) 中將要執行的指令載入到 CPU 的指令暫存器 (IR) 中，同時程式計數器 (PC) 會加 1 以指向下一道指令。
2. **解碼 (Decode)**：控制單元 (CU) 解析指令暫存器中的內容，判斷這是一道什麼指令（加法、移動、跳躍等），以及資料在哪裡。
3. **執行 (Execute)**：算術邏輯單元 (ALU) 或其他元件實際執行該指令要求的運算操作。
4. **儲存 (Store / Write-back)**：將執行算完的最終結果，寫回到 CPU 內部的暫存器或主記憶體中。

## 常考陷阱 / 易混淆點

> ⚠️ 特別注意：
> 1. **步驟順序**：絕對不能背反！永遠是 `擷取 → 解碼 → 執行 → 儲存`。
> 2. **負責的單元不同**：
>    - 負責「擷取下一個指令位置」：程式計數器 (PC)
>    - 負責「解碼」：控制單元 (CU)
>    - 負責「算術與邏輯執行」：算術邏輯單元 (ALU)

> 💡 類比：餐廳廚師做菜
> 1. 拿訂單（擷取 Fetch）
> 2. 看懂訂單上的菜名與備料（解碼 Decode）
> 3. 開火炒菜（執行 Execute）
> 4. 把菜裝盤放上出餐區（儲存 Store）
