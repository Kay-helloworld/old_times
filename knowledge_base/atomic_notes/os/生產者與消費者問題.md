# 生產者與消費者問題

## 核心定義

> 一句話定義：這是作業系統中處理「多執行緒同步 (Synchronization)」的經典問題。一個生產資料放入緩衝區 (Buffer)，另一個取出資料消費；當緩衝區滿了，生產者必須等待；當緩衝區空了，消費者必須等待。

## 詳細說明

在作業系統中，有兩個（或多個）行程共享一塊固定大小的記憶體緩衝區（Bounded-Buffer）。

1. **生產者 (Producer)** 的工作是產生資料，把它放入緩衝區。
2. **消費者 (Consumer)** 的工作是從緩衝區拿走資料，然後處理。
3. **發生衝突的原因 (Race Condition)**：因為緩衝區是共享的變數 `count`，如果兩人同時想修改（放進去/拿出來），就會造成資料不一致。

為了避免錯誤，必須使用**同步機制（如：號誌 Semaphore 或 互斥鎖 Mutex）**來協調：
- 【**緩衝區已滿時**】：生產者不能再放資料，必須**暫時進入等待狀態 (Wait / Block / Sleep)**。直到消費者拿走一個東西，發出通知 (Wake up / Signal) 叫醒生產者。
- 【**緩衝區空位時**】：同理，消費者不能拿空氣，必須**進入等待 (Wait)**，直到生產者放入一個東西，再把它叫醒 (Signal)。

## 常考陷阱 / 易混淆點

> ⚠️ 特別注意：
> 1. **等待的方法選擇**：考試會故意混淆。生產者看到緩衝區滿了，**不能「結束程式執行」或「直接拋出異常退出」**，正確的行為是：呼叫等待指令（例如 `wait()` 或 `pthread_cond_wait()`）來自我休眠。
> 2. **覆蓋資料**：生產者在滿了的時候**絕對不能覆蓋舊資料**，因為那是未消費的寶貴資料！

## 備考難度

難度估計：⭐⭐⭐

- 初階選擇題通常考觀念（滿了怎麼辦？空了怎麼辦？），只要選「等待 / 睡眠（Sleep、Wait）」。
- 進階選擇題會考 C 語言的 `pthread` 或號誌 (`P/V` 或 `Wait/Signal`) 的寫法，會稍微偏硬一點。

## 快速記憶

> 💡 類比：壽司店迴轉台（緩衝區）
> - **壽司師傅（生產者）**：看到台上放滿盤子了，不能硬塞（不能覆蓋舊壽司），更不能直接下班回家（不能結束程式），只能去後面抽根菸**睡覺/等待 (`wait`)**。
> - **客人（消費者）**：看到台上沒壽司了，不能吃空氣，只能喝茶**等待 (`wait`)**，等師傅捏好新壽司放上去，叫他一聲（`signal`），他才能吃。
