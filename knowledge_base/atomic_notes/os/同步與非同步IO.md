# 同步與非同步 I/O

## 核心定義

> 一句話定義：
> - **同步 I/O (Synchronous I/O)**：程式發出 I/O 請求後，必須**停下來等待**（被阻塞 Blocking），直到 I/O 動作完成才能繼續往下執行。
> - **非同步 I/O (Asynchronous I/O)**：程式發出 I/O 請求後，**不需等待**，可以立刻繼續往下執行其他工作，I/O 完成時會透過中斷 (Interrupt) 或回呼函數 (Callback) 來通知程式。

## 詳細說明

在作業系統中，CPU 的運算速度遠大於磁碟、網路等 I/O 設備。因此程式與 I/O 互動的方式深深影響系統效能。

1. **同步 I/O 的特性：**
   - 流程簡單直觀，程式碼容易撰寫與除錯。
   - 缺點是 CPU 在等待 I/O 期間處於閒置狀態（如果是單一執行緒的話），這會導致資源浪費。
   - 考試常以「阻塞 (Blocking)」一詞來描述同步 I/O 讓程式停下來的行徑。

2. **非同步 I/O 的特性：**
   - 能夠讓 CPU 與 I/O 設備平行運作（Overlap），大幅提升系統整體吞吐量 (Throughput)。
   - 缺點是程式碼變得複雜，因為需要處理事件通知 (Event Notification) 或中斷。

## 常考陷阱 / 易混淆點

> ⚠️ 特別注意：
> 1. **「同步」聽起來像是一起進行，其實在 OS 中是指「互相牽制、必須等待」**。不要跟字面的意思搞混了！
> 2. 考題若問「哪種方式會使程式暫時停下來等待？」答案一定是 **同步 I/O** (或稱阻塞 Blocking I/O)。

## 備考難度

難度估計：⭐

- 常識記憶題。只要釐清「同步 = 死等，非同步 = 發射後不理(事後通知)」的觀念，這類題目就是秒殺題。

## 快速記憶

> 💡 類比：買手搖飲料
> - **同步 I/O**：點完餐後，你站在櫃檯前面**死死盯著**店員做飲料，什麼都不做，直到飲料做好拿給你才離開去辦下一件事。
> - **非同步 I/O**：點完餐後，店員給你一個**取餐呼叫器**，你轉頭走去隔壁買雞排（做其他事），等呼叫器震動（Interrupt/Callback）時，你再回來拿飲料。
