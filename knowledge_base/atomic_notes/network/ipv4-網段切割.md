# IPv4網段切割

## 核心定義

> 一句話定義：將一個大的 IPv4 網路（依據 ABC Class）利用「子網路遮罩」的位元借位，劃分成多個較小的「子網路」，以減少廣播風暴並節省 IP 資源。

## 詳細說明

IPv4 位址由 32 個位元 (bits) 組成，分為 4 個 8位元組 (octets)。
在進行網段切割（Subnetting）時，我們向**主機識別碼 (Host ID) 借用位元**，變成**子網路識別碼 (Subnet ID)**。

切割公式兩大原則：
1. **子網路數量 (Subnets)** = `2^n` （n 為借用的位元數）
2. **每個子網路的主機數量 (Hosts)** = `2^h - 2` （h 為剩下的主機位元數，減去的 2 是因為全為 0 的是 Network ID，全為 1 的是 Broadcast ID）

例如：一個 `/24` (255.255.255.0) 的 Class C 網段向主機位元借 2 個 bit 變成 `/26`：
- 子網路數：`2^2 = 4` 個子網路
- 每個子網路可用主機：`2^6 - 2 = 62` 台主機

## 常考陷阱 / 易混淆點

> ⚠️ 特別注意：
> 1. **可用 IP 數 vs 總 IP 數**：考試常問「可配置/可用 (usable) 給主機的 IP 個數有幾個？」，一定要記得 **減去 2**（Network ID 跟 Broadcast ID 不能指派給電腦）。若只問「總 IP 個數」則不用減。
> 2. **找出廣播位址**：給定一個 IP 和遮罩，廣播位址就是把主機部分（Host 欄位）的二進位「全部填滿 1」。
> 3. **遮罩寫法轉換**：255.255.255.192 就是 `/26` (因為 192的二進位是 11000000，代表在最後一個 octet 前面有兩個 1，加上前三組全滿的 24 個 1 = 26)。常用的數字必須超級熟：128(/25), 192(/26), 224(/27), 240(/28), 248(/29), 252(/30)。

## 備考難度

難度估計：⭐⭐⭐

- 需要實際動筆計算二進位，但只要把 2 的次方表與遮罩數字（128, 192, 224...）背熟，就是送分題。經濟部這科一考通常是好幾題連發。

## 快速記憶

> 💡 解題必備對照表：向右看幾個 bit 借位
> `/24` = 255.255.255.0 (可用主機 254)
> `/25` = .128 (主機 126, 跳號 128)
> `/26` = .192 (主機  62, 跳號 64)
> `/27` = .224 (主機  30, 跳號 32)
> `/28` = .240 (主機  14, 跳號 16)
> `/29` = .248 (主機   6, 跳號 8)
> `/30` = .252 (主機   2, 跳號 4，常作路由器間點對點連線點)
